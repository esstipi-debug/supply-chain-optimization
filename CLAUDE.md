## What Linchpin is

An **agentic supply-chain engine**: a plain-language brief becomes a finished,
QA-gated deliverable (Excel + report + chart), grounded in a knowledge graph of
23 SCM books and the codebase itself. A deterministic Python **engine** (EOQ,
safety stock, `(s,Q)`/`(R,S)`, forecasting, pricing, DDMRP, ABC-XYZ, sourcing,
landed cost, cost-to-serve, S&OP…) sits under an **orchestrator agent** that
classifies → runs → validates → delivers. Runs with or without an LLM.

Positioning: **the human sells & decides, Linchpin produces 10×.** Every
consequential result ends in one of four outcomes — and three need a human
(see *Never-unprotected* below).

## How the agent works

`brief → intent.classify → registry.get(tool) → prepare → run → QA → deliver → JobResult`

- **"QA fails ⇒ no deliverable"** is enforced in one place (the orchestrator).
- Statuses: `ok` · `needs_clarification` · `needs_data` · `qa_failed` · `error`.
- Adding a capability = one `register()` call + a `jobs/<x>_job.py`; no routing edits.

### Never-unprotected (`src/guided.py`)
Every consequential result is a `GuidedOutcome` with status:
`EXECUTED` (agent did it) · `OPTIONS` (ranked choices) · `HANDOFF` (prepared
human step) · `ESCALATED` (routed to a human with SLA). The last three need a
human — that contract is what `documentation/operator/` documents.

### Writeback safety (`src/writeback.py`)
Never mutate a system of record blindly: stage a dry-run `Changeset`, classify by
risk tier (`read`/`reversible`/`irreversible`), gate on a time-boxed `Approval`
(900 s TTL), apply idempotently, `rollback()`-able. Irreversible always needs
explicit approval.

## Dev conventions

- **Python 3.11+**. Tests: `pytest tests/ -q` with `PYTHONPATH=.` (the hook persists it).
- **Lint scope** (matches CI): `ruff check src tests examples`.
- The **SessionStart hook** (`.claude/hooks/session-start.sh`) auto-installs
  `requirements-dev.txt` (web only) and refreshes `graphify-out/`. It's idempotent.
- **Workflow:** feature branch → **draft PR** → CI green (`tests` on py3.11/3.12/3.13)
  → **squash-merge** to `main`. Never push straight to `main`.
- New agent tool recipe: `jobs/<x>_job.py` with a pandas-only `prepare()` (read its
  own CSV, **not** `intake.py`) → `run`/`verify` → a `Tool` in `scm_agent/tools.py`
  with distinctive multi-word `intent_keywords`.

## Gotchas (read before committing)

- **ASCII-only in console prints** (Windows cp1252 breaks on em dashes). Markdown
  files written utf-8 are fine.
- **Gitignored & regenerable:** `graphify-out/`, `data/`, `deliverables/`, `.env`.
  Don't commit them. `.claude/settings.json` + `.claude/hooks/` **are** versioned;
  `.claude/settings.local.json` is personal.
- **Never read or surface PII** — some datasets (e.g. DataCo) carry customer PII;
  analysis is aggregate-only.
- **Don't paste secrets** (API keys) into chat or commits.
- The **books graph** (L3) needs an LLM backend (Kimi/Gemini) to rebuild; the
  **code graph** is AST-only (no key) and the hook handles it.
